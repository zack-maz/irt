# Phase 23.1: Detail Panel Navigation Stack - Context

**Gathered:** 2026-04-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Add browser-like navigation history to the detail panel. Every entity or cluster viewed is pushed onto a stack. Users can navigate back through their history via a breadcrumb row with directional slide transitions. Closing the panel clears the stack.

</domain>

<decisions>
## Implementation Decisions

### Navigation Scope
- Universal history — every entity click (map or panel drill-down) pushes onto the stack
- Map clicks push onto the existing stack (not a fresh start)
- Unlimited depth, no cap on stack entries
- Closing the panel (× button or Escape key) clears the entire stack
- No forward navigation button — back-only with discard-forward-history (browser fork behavior)

### Stack Behavior
- Stack entries are generic "panel view" objects — no typed distinction between clusters, flights, ships, events, sites
- Lost-contact entities remain in the stack — navigating back shows them with existing LOST CONTACT banner and grayscale treatment
- No depth count badge on the back button — just a plain arrow

### Breadcrumb Row
- Thin row below the header, appears on demand when stack has entries (not always reserved)
- Shows previous entries only — current view is NOT shown in the breadcrumb (header already displays it)
- Labels: type + name, truncated with ellipsis for long names (e.g., "Cluster(5)" > "Airstrike" > "IR-721")
- Styling: uniform muted gray text, no entity color dots, `>` chevron separators
- Interaction: ← arrow goes back one step; last (most recent) segment is also clickable; deeper segments are display-only
- Overflow: horizontal scroll when breadcrumb exceeds panel width (360px)
- Entrance/exit: slides in/out vertically when first entry pushed / stack emptied

### Transitions
- Directional slide between views: drill-in slides content left (new from right), back slides content right (previous from left)
- Duration: 150ms (fast, snappy)
- Map fly-to and panel slide transition happen simultaneously (no waiting)

### Claude's Discretion
- Exact easing curve for slide transitions
- Stack data structure implementation (array in uiStore vs. separate store)
- How to snapshot cluster state for stack entries (store full cluster object vs. ID + re-derive)
- Breadcrumb row height and typography sizing
- Truncation threshold for breadcrumb segment names

</decisions>

## Existing Code Insights

### Reusable Assets
- `src/components/layout/DetailPanelSlot.tsx`: Current panel with cluster/entity branching — primary file to modify
- `src/components/detail/ThreatClusterDetail.tsx`: Cluster drill-down that calls `selectEntity` (loses cluster context)
- `src/hooks/useSelectedEntity.ts`: Cross-store entity lookup with lost contact tracking via useRef
- `src/stores/uiStore.ts`: `selectedEntityId` and `selectedCluster` with mutual exclusion (`selectEntity` clears cluster, `setSelectedCluster` clears entity)

### Established Patterns
- Panel open/close: 300ms CSS transform `translate-x` transition on DetailPanelSlot
- Entity selection: `selectEntity(id)` + `openDetailPanel()` pattern from BaseMap onClick
- Cluster selection: `setSelectedCluster(cluster)` clears `selectedEntityId` (mutual exclusion)
- Lost contact: `useSelectedEntity` holds last-known entity in useRef, returns `isLost: true`

### Integration Points
- `uiStore.ts` — needs navigation stack state (array of panel views) and push/pop/clear actions
- `DetailPanelSlot.tsx` — needs breadcrumb row component and directional slide transition wrapper
- `ThreatClusterDetail.tsx` — `handleEventClick` should push current cluster view before selecting event
- `BaseMap.tsx` — onClick handler should push current view before selecting new entity
- All fly-to callers (notifications, counters, search) should push current view when opening a new entity

</code_context>

<specifics>
## Specific Ideas

- Breadcrumb should feel like iOS navigation push/pop — directional slide gives spatial sense of depth
- 150ms is intentionally faster than the 300ms panel open/close — you're already inside the panel
- Breadcrumb excludes current view because the header already shows it — no redundancy
- Horizontal scroll preferred over truncation-from-left for overflow — preserves full history access

</specifics>
