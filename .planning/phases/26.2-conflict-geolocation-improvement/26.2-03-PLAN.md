2. Define ACTOR_COUNTRY_MAP (NLP-extracted actor keywords -> expected FIPS codes):
```typescript
const ACTOR_COUNTRY_MAP: Record<string, string[]> = {
  'iran': ['IR'], 'iranian': ['IR'], 'tehran': ['IR'], 'irgc': ['IR'], 'quds': ['IR'],
  'israel': ['IS'], 'israeli': ['IS'], 'idf': ['IS'], 'mossad': ['IS'],
  'iraq': ['IZ'], 'iraqi': ['IZ'], 'baghdad': ['IZ'],
  'syria': ['SY'], 'syrian': ['SY'], 'damascus': ['SY'], 'assad': ['SY'],

  'hezbollah': ['LE', 'SY'], // Lebanon/Syria
  'houthi': ['YM'],
  'taliban': ['AF', 'PK'],
  'pkk': ['TU', 'IZ', 'SY'], // Kurdish, cross-border
  // ... extend as appropriate
};
```

3. Export `NlpValidationResult` type:
```typescript
export type NlpValidationResult =
  | { status: 'verified' }                                                    // NLP confirms location
  | { status: 'mismatch'; reason: string }                                    // NLP contradicts -> reject
  | { status: 'relocated'; newLat: number; newLng: number; cityName: string } // Centroid moved to real city
  | { status: 'skipped'; reason: string };                                    // NLP couldn't determine (title fail, no data)
```

4. Export `validateEventGeo(params: { title: string | null; actorCountryCodes: { actor1: string; actor2: string }; geoCountryCode: string; actionGeoType: number | undefined; lat: number; lng: number; }): NlpValidationResult`:

Logic:
a. If title is null -> return { status: 'skipped', reason: 'title_fetch_failed' }
b. Call extractActorsAndPlaces(title) for NLP extraction
c. Collect all actor signals:
   - From GDELT Actor1CountryCode/Actor2CountryCode: convert via CAMEO_TO_FIPS
   - From NLP-extracted actors: look up each word in ACTOR_COUNTRY_MAP
   - From NLP-extracted places: look up each place via lookupCityCoords for its countryCode
d. Collect expected FIPS codes from actor signals (deduplicated set)
e. If no actor signals at all (both GDELT fields empty AND NLP found nothing) -> return { status: 'skipped', reason: 'no_actor_data' }
f. Cross-validate: Check if geoCountryCode appears in the set of expected countries from actor signals OR from NLP place lookups.

CRITICAL for cross-border events: If NLP extracted places and ANY extracted place's country matches geoCountryCode, the event is valid. The actor's country does NOT need to match -- "Israel strikes Damascus" is geocoded to Syria, and Syria is the correct location because NLP extracted "Damascus" which maps to SY.

g. If NLP extracted at least one place AND the place's country matches geoCountryCode -> { status: 'verified' }
h. If NLP extracted places but NONE match geoCountryCode AND actor countries don't include geoCountryCode -> { status: 'mismatch', reason: description }
i. If actionGeoType is 3 or 4 (centroid/landmark) AND NLP extracted a specific city name that can be looked up -> { status: 'relocated', newLat, newLng, cityName } using lookupCityCoords with original lat/lng for disambiguation
j. Otherwise if actor country codes include geoCountryCode -> { status: 'verified' }
k. If only actor countries found and NONE match geo -> { status: 'mismatch' }
l. Default fallback -> { status: 'skipped', reason: 'insufficient_signal' }
