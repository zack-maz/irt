const CONCURRENCY = 10;

   - Keep keys short to save Redis memory

  <files>server/lib/titleFetcher.ts, server/__tests__/lib/titleFetcher.test.ts</files>
  <behavior>
    - batchFetchTitles returns Map of URL to title for successful fetches
    - batchFetchTitles returns null value for URLs that fail to fetch
    - batchFetchTitles checks Redis cache first and skips HTTP fetch for cached URLs
    - batchFetchTitles caches successful title extractions in Redis
    - batchFetchTitles handles empty URL array (returns empty Map)
    - batchFetchTitles does not cache null/failed results in Redis
    - batchFetchTitles processes URLs in batches of CONCURRENCY (10) at a time
  </behavior>
  <action>
Add to `server/lib/titleFetcher.ts`:

1. `batchFetchTitles(urls: string[]): Promise<Map<string, string | null>>`:
   - Deduplicate input URLs
   - Check Redis cache first for each URL: `cacheGet<string>(CACHE_PREFIX + hashUrl(url), LOGICAL_TTL_MS)`
   - Collect uncached URLs
   - Fetch uncached in batches of CONCURRENCY using `Promise.allSettled`:
     - Each fetch: `fetch(url, { signal: AbortSignal.timeout(FETCH_TIMEOUT_MS), headers: { 'User-Agent': 'IranConflictMonitor/1.0' } })`
     - Read response body incrementally up to MAX_BYTES (10KB), then cancel the reader
     - Parse title via extractTitleFromHtml
     - On non-2xx response or timeout: return null for that URL
   - Cache successful extractions (non-null titles) in Redis: `cacheSet(CACHE_PREFIX + hashUrl(url), title, REDIS_TTL_SEC)`
   - Do NOT cache null results (failed fetches may succeed later)
   - Return complete Map<url, title | null>

2. Handle edge cases:
   - Empty/malformed URLs: skip fetch, set null
   - SOURCEURL with multiple URLs (semicolon or pipe separated): caller should split before calling -- but defend against it by taking first URL-like substring
   - Abort reader after MAX_BYTES to avoid downloading full articles

- Cache hit returns cached title without fetch
- Cache miss triggers fetch and caches result
- Failed fetch returns null and does not cache
- Empty URL array returns empty Map
- Batch respects concurrency (verify fetch is called correct number of times)
