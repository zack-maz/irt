# Phase 27.3.1 UAT Gap Diagnosis

Consolidated output of 5 parallel `gsd-debugger` investigations spawned from `27.3.1-UAT.md`. These findings feed the `gsd-planner` gap-closure phase.

## G1 ‚Äî "Dam near X" generic labels still admitted (major)

**Hypothesis CONFIRMED.** `hasName()` at `server/adapters/overpass-water.ts:133-138` short-circuits on `isNotable(tags)` ‚Äî any element with `wikidata`/`wikipedia`/`wikipedia:lang` tags passes hasName without a real name. Downstream: `extractLabel` at `:390-397` only accepts Latin-script names, returns `FACILITY_TYPE_LABELS[facilityType]` ("Dam") otherwise. `server/lib/waterLabeling.ts:37` matches these bare generics via `GENERIC_LABELS` set and hits Nominatim to rewrite as "Dam near \<city|country\>".

**UAT match:** `unnamed: 139, geocoding: 139` ‚Äî 139 facilities cleared `hasName` via wikidata but lacked Latin-script names.

**Recommended fix:**

1. Tighten `hasName` to real-name only:
   ```typescript
   export function hasName(tags): boolean {
     if (tags['name']?.trim()) return true;
     if (tags['name:en']?.trim()) return true;
     if (tags['operator']?.trim()) return true;
     return false;
   }
   ```
2. `computeAdmissionDecision` (line 665) needs NO structural change ‚Äî the `no_name` bucket will grow from ~0 to ~139.
3. **Delete `server/lib/waterLabeling.ts`** + callers in `server/routes/water.ts` + `scripts/refresh-water-facilities.ts`. After tightening, no admitted facility reaches the generic fallback. (Plan 06 D-10 already flagged this.)
4. `src/lib/waterLabel.ts` `GENERIC_TYPE_RE` still useful for non-Latin-only named facilities (e.g. Persian `name:fa` only) ‚Äî keep with updated comment.

**Expected admission drop:** ~139 facilities (23% of current 602) ‚Üí ~463 total.

**Files:** `server/adapters/overpass-water.ts`, `server/lib/waterLabeling.ts` (delete), `server/routes/water.ts`, `scripts/refresh-water-facilities.ts`, `server/__tests__/adapters/overpass-water.test.ts`, `src/lib/waterLabel.ts` (comment only).

**Expected total after G1 + G2:** ~437 facilities (dominated by Iran / Iraq / Syria / Saudi / UAE / Afghanistan / Egypt). ~130 of the 165 Turkey admits overlap with the 139 wikidata-only admits (same root cause), so G1 + G2 combined drop ~174 net.

## G3 + G4 ‚Äî Water + Sites observability zeros on Redis hit (major)

**Hypothesis CONFIRMED.** `cacheSetSafe` at `server/routes/water.ts:169,201,218,279` and `server/routes/sites.ts:95,111` stores bare `WaterFacility[]` / `SiteEntity[]`. `filterStats` envelope is never persisted. On cache-hit (water.ts:149-159, sites.ts:82-87), the route unconditionally calls `buildEmptyFilterStats` ‚Äî producing zeros. Plan 03 SUMMARY ¬ß"Route response paths √ó source attribution" documents this as a known-accepted gap at the time.

**Recommended fix ‚Äî Option A (cache the full envelope):**

1. Introduce internal payload types:
   ```typescript
   type WaterCachePayload = { facilities: WaterFacility[]; filterStats: WaterFilterStats };
   type SitesCachePayload = { sites: SiteEntity[]; filterStats: SiteFilterStats };
   ```
2. Every `cacheSetSafe` call site (4 in water, 2 in sites) stores the payload object instead of the bare array.
3. On cache-hit: spread cached `filterStats` BUT overwrite `source: 'redis'` and `generatedAt: new Date(cached.lastFresh).toISOString()`. **Critical** ‚Äî otherwise Redis echoes `'snapshot'` because snapshot tier seeded the stats.
4. Drop `buildEmptyFilterStats` on cache-hit branches; retain only for the error-without-cache fallback branch.
5. `/api/water/precip` reuse path (water.ts:273-279) currently reads `cachedFacilities.data` as bare array ‚Äî MUST unwrap `.facilities`.
6. **Bump Redis keys** (`water:facilities` ‚Üí `water:facilities:v2`, `sites:v2` ‚Üí `sites:v3`) to force cold-start after deploy. Cleaner than runtime shape-guard code.

**Schema/Zod:** `waterFilterStatsSchema` + `siteFilterStatsSchema` validate the RESPONSE envelope, not the Redis storage format. No Zod changes needed. Truth 21 (Plan 03) still holds ‚Äî outer response never omits `filterStats`.

**Memcache fallback** (`redis.ts:101-108`) round-trips same shape ‚Äî no extra work but cover in tests.

**Files:** `server/routes/water.ts`, `server/routes/sites.ts`, optional tiny changes in `server/cache/redis.ts` (types only), `server/__tests__/routes/water.test.ts`, `server/__tests__/routes/sites.test.ts`.

## G5 ‚Äî `npm run refresh:water` hangs on Redis env missing (major)

**Hypothesis CONFIRMED trivially.** `package.json` `refresh:water` = `tsx scripts/refresh-water-facilities.ts` ‚Äî no env flag. Script imports `overpass-water.js` ‚Üí `redis.ts:5-8` instantiates Upstash client at module-load reading `process.env.UPSTASH_REDIS_REST_URL/_TOKEN` (both `undefined` without env). Every `cacheGet/cacheSet` hits the 2000ms `withTimeout` fallback ‚Üí 139 facilities √ó 2 ops √ó 2s ‚âà 9 min of hangs.

**Recommended fix (1-liner √ó 2, mirrors `dev:server` at package.json:12):**

```json
"refresh:water": "node --env-file-if-exists=.env --import tsx/esm scripts/refresh-water-facilities.ts",
"refresh:sites": "node --env-file-if-exists=.env --import tsx/esm scripts/refresh-sites.ts",
```

`extract-ethnic-data.ts` / `extract-rivers.ts` / `extract-aqueduct-basins.ts` don't need the fix ‚Äî they don't import `server/cache/redis.ts`.

**Files:** `package.json` only. No source code changes.
