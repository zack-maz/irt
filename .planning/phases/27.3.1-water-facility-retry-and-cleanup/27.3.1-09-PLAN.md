<objective>
Close UAT gap G5 (major): `npm run refresh:water` hangs mid-geocoding under error
floods of `[Upstash Redis] Redis client was initialized without url or token.
Failed to execute command.`, forcing the user to kill the process and blocking
snapshot regeneration. Root cause (confirmed in 27.3.1-DIAGNOSIS.md §G5): the
`refresh:water` + `refresh:sites` npm scripts use plain `tsx scripts/...`
without `--env-file-if-exists=.env`, so the Upstash client instantiates at
module-load time with `undefined` URL/TOKEN. Every `cacheGet/cacheSet` call then
hits the 2000ms `REDIS_OP_TIMEOUT_MS` `withTimeout` Promise.race fallback and
fails — the refresh script does 139 reverse-geocode calls × 2 cache ops × 2s
timeout ≈ 9 min of hung time.

Purpose: Unblock snapshot regeneration. Plan 10 (G1 + G2 admission hardening)
MUST regen `src/data/water-facilities.json` after tightening the admission gate,
and it cannot do that unless this trivial 1-liner × 2 ships first. Sites script
has the same bug but the UAT didn't reproduce because the sites adapter has no
reverse-geocoding phase (Test 7 passed) — still a latent defect worth fixing at
the same time for symmetric fix semantics.

Output:

- `package.json` `scripts.refresh:water` updated to `node --env-file-if-exists=.env --import tsx/esm scripts/refresh-water-facilities.ts`
- `package.json` `scripts.refresh:sites` updated to the parallel pattern
- Verification: a `refresh:water` run completes without emitting any
  `[Upstash Redis] ... Failed to execute command` lines (instead Redis cache
  hits populate from the 30-day geocode cache like the dev server does)

This plan is intentionally the smallest thing that unblocks Plan 10. No code
changes, no tests added beyond the smoke verification below — G5 is a
configuration fix, not a behavior change.

</objective>

From server/cache/redis.ts (top of file — instantiated at module-load):

```typescript
import { Redis } from '@upstash/redis';
export const redis = new Redis({
  url: process.env.UPSTASH_REDIS_REST_URL,
  token: process.env.UPSTASH_REDIS_REST_TOKEN,
});
// withTimeout Promise.race wraps every cache call at REDIS_OP_TIMEOUT_MS=2000
```

The script import chain: `scripts/refresh-water-facilities.ts` →
`server/adapters/overpass-water.ts` → (no direct redis import, but the transitive
`server/lib/waterLabeling.ts` import triggers `server/cache/redis.ts` at top of
file). So the moment the script's imports resolve, the Upstash client is already
instantiated with `undefined` URL.

Fix shape (mirrors `dev:server` exactly — line 12):

```json
"refresh:water": "node --env-file-if-exists=.env --import tsx/esm scripts/refresh-water-facilities.ts",
"refresh:sites": "node --env-file-if-exists=.env --import tsx/esm scripts/refresh-sites.ts",
```

`--env-file-if-exists` (Node ≥20.12) is non-fatal when `.env` is absent. The
engines field already pins `"node": "22.x"`, so the flag is available.
