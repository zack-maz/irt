# Plan 09 Summary — G5 env-load fix for refresh scripts

## Outcome

Closed UAT gap G5 (major). `npm run refresh:water` and `npm run refresh:sites`
now boot Node with `--env-file-if-exists=.env --import tsx/esm`, mirroring the
existing `dev:server` pattern on package.json line 12.

Before the fix the scripts ran as bare `tsx scripts/...`, so the import chain
resolved `server/cache/redis.ts` before any `.env` was loaded. The Upstash
client instantiated with `undefined` URL/TOKEN, and every subsequent
`cacheGet`/`cacheSet` hit the 2000ms `withTimeout` Promise.race fallback —
~139 reverse-geocode calls × 2 cache ops × 2s ≈ 9 min of cascading Redis
timeout errors the user had to kill.
