Lebanon+Israel+Yemen). Root causes fully confirmed in
`27.3.1-DIAGNOSIS.md` §G1 + §G2.

Strategy per the diagnosis:

- **G1 fix:** tighten `hasName()` in `server/adapters/overpass-water.ts` to
  require a real `name` / `name:en` / `operator` tag. Remove the
  `if (isNotable(tags)) return true` short-circuit that let wikidata-only
  facilities pass D-05 without a real name. Downstream: the `no_name` bucket
  grows from ~0 to ~139, and since no admitted facility has a generic type
  label anymore, `labelUnnamedFacilities` becomes dead code and its entire
  server/lib/waterLabeling.ts file can be deleted (CONTEXT.md D-10 already
  flagged this conditional on D-05 holding — post-G1 it finally holds).
- **G2 fix:** drop Turkey from `PRIORITY_COUNTRIES`. Add an `excluded_turkey`
  rejection bucket to `WaterFilterStats.rejections` + `byTypeRejections`
  (mirrors the sites adapter which already has this bucket — Test 7 confirmed
  it excluded 156 Turkey sites). Extend `computeAdmissionDecision` with a
  new reject branch: `if (nearestCountryName(lat,lng) === 'Turkey') reject`.
  The branch sits AFTER `excluded_location` (which catches western Turkey by
  the 600km-from-Diyarbakir rule) so the buckets stay semantically distinct:
  `excluded_location` = geographic rule, `excluded_turkey` = country-exclusion
  rule.
- **Snapshot regen:** after the code changes land, run `npm run refresh:water`
  (unblocked by Plan 09). Expected new count ~437 facilities (evidence in
  DIAGNOSIS §G2: "G1 + G2 combined drop ~174 net" from 602 → ~428). Commit the
  regenerated JSON in the same plan so the plan ships a consistent admission
  rule + matching snapshot. Also regenerate sites snapshot as a regression
  guard (sites was PASS in UAT Test 7 but refresh:sites env bug could hide a
  latent defect) — OPTIONAL if operator short on time.

`PRIORITY_COUNTRIES` as a tangential expansion recommendation. That is NOT in
the UAT evidence for G1 or G2 (the UAT only flagged Turkey over-representation,
not Pakistan under-representation), so Pakistan expansion is explicitly OUT of
scope for this plan. If Pakistan coverage proves thin in post-regen review,
file a separate backlog note — do not bundle it with Turkey removal.
