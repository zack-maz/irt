<objective>
Close UAT gaps G3 (Water Filters DevApiStatus panel all-zeros on Redis hit)
and G4 (Sites Filters DevApiStatus panel all-zeros on Redis hit). Single root
cause per 27.3.1-DIAGNOSIS.md §G3+G4: the route Redis write paths store a
BARE facilities/sites array, losing the `filterStats` envelope. On every cache
hit after Redis warms, `buildEmptyFilterStats` synthesizes a zero-tally
placeholder for the response — so DevApiStatus renders zeros for byCountry,
byTypeRejections, Overpass health, and per-type counts. R-08 observability is
effectively dead once Redis has been warmed (10s after cold start).

Fix strategy per DIAGNOSIS Option A ("cache the full envelope"):

1. Introduce internal payload types:
   ```typescript
   // water.ts
   type WaterCachePayload = { facilities: WaterFacility[]; filterStats: WaterFilterStats };
   // sites.ts
   type SitesCachePayload = { sites: SiteEntity[]; filterStats: SiteFilterStats };
   ```
2. All 4 cacheSetSafe call sites in water.ts + 2 in sites.ts store the payload
   object, not bare arrays.
3. On cache-hit, spread `cached.data.filterStats` into the response AND
   overwrite `source: 'redis'` + `generatedAt: ISO(cached.lastFresh)`. The
   overwrite is CRITICAL — without it, Redis echoes the snapshot's
   `source: 'snapshot'` because the snapshot tier populated Redis. The
   response would then misrepresent provenance.
4. Drop `buildEmptyFilterStats` on cache-hit branches; retain only for the
   error-without-cache branch (where no real stats exist because
   `fetchWaterFacilities`/`fetchSites` threw).
5. `/api/water/precip` at lines 273-279 reads `cachedFacilities.data` as a
   bare array today. Post-envelope, unwrap `.facilities`:
   ```typescript
   facilities = cachedFacilities.data.facilities; // was: cachedFacilities.data
   ```
6. Redis keys bump to force a cold fill on deploy:
   - `water:facilities` → `water:facilities:v2`
   - `sites:v2` → `sites:v3`
     Old keys become orphaned, expire via 24h logical / 72h hard TTL. No
     runtime shape-guard code needed — the bump ensures the first read after
     deploy is a miss, falls through to snapshot, and writes the new envelope
     shape.

Purpose: Restores R-08 observability end-to-end. Once Redis warms from the
snapshot (Plan 05 tier behavior), the byCountry table, byTypeRejections, and
per-type counts all round-trip through the cache instead of being blanked
out. DevApiStatus becomes usable again.

This plan runs in Wave 3 AFTER Plan 10 (Wave 2). Plan 10 deletes
`labelUnnamedFacilities` and therefore changes the exact code shape at each
cacheSetSafe write site in water.ts (no more `labeled` intermediate variable).
Plan 11's Step 3 action text assumes the post-Plan-10 shape. Serializing
Plan 10 → Plan 11 also resolves the `server/routes/water.ts` files_modified
overlap cleanly — no shared-file wave parallelism headache.

Plan 11 does NOT regenerate snapshots (that's Plan 10 Task 4). Plan 11
assumes the snapshot file (post-Plan-10) has a populated `stats`
sub-object — which is true per Plan 05's refresh script.

Output:

- `server/routes/water.ts`: WaterCachePayload type, 4 cache-store envelope
  writes, 4 cache-hit envelope reads, key bump to v2, /api/water/precip
  unwrap fix
- `server/routes/sites.ts`: SitesCachePayload type, 2 cache-store envelope
  writes, 2 cache-hit envelope reads, key bump to v3
- Route tests updated to cover the envelope round-trip
- PR description / deploy note documenting the Redis key orphaning
