## Decisions Made

- **no_city scope = reservoirs only.** The phase goal "reduce reservoir noise to 50-200 via HOLISTIC filter" was the original rationale for no_city; dams and desalination should never have been in scope. Plan 05 narrows the rule to the category it was actually meant to prune.
- **Named-priority exemption** layered on top. A named reservoir in Iran/Iraq/Syria/Afghanistan/etc. is intelligence-relevant even without a CITY_DATA city within 150km — the exemption protects these cases.
- **Name override is one-directional.** `classifyWaterType` only promotes reservoir → dam when `/\bdam\b/i` matches; it never demotes dams or desalination, preventing adversarial OSM naming from recategorizing confirmed infrastructure.
- **Whole-string sentinel** (not substring). "Mosul Dam" must survive; only bare "Dam" (trimmed, case-insensitive) is treated as missing.
- **Placeholder over cache-shape change.** Rendering the placeholder client-side avoids extending the server cache schema to persist filterStats — smaller blast radius.

## Deviations from Plan

**Minor — Test assertion widened.** The Plan 05 rewrite of the Plan 04 "rejects priority-country dam" test (now testing "rejects unnamed non-priority reservoir") initially asserted `rejections.no_city + rejections.low_score >= 1`, but the REV-2 `not_notable` gate fires first for unnamed/un-wikidata'd reservoirs. Widened the assertion to `no_city + low_score + not_notable >= 1` — semantically correct (the reservoir must be rejected by SOME gate; which gate catches first is an implementation detail). Similarly, the new "keep desalination no nearestCity" test required an `operator: 'ADNOC'` tag to push the notability score above MIN_NOTABILITY_SCORE=25 so the test exercises the no_city exemption, not the low_score gate. Both adjustments are in test code only and preserve the intended behavioral invariants.
