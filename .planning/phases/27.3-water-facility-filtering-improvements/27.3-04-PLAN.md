<objective>
Close the final Phase 27.3 UAT gap: Test 3 — users see the fallback label "Dam near unknown" on low-information water facilities. The detail panel renders correctly, so the bug is entirely in the LABELING path, not the rendering path.

**Root cause (ground-truthed in server/routes/water.ts lines 64 + 73):** the server-side `labelUnnamedFacilities` function ALREADY writes the literal string `'Unknown'` into `facility.label` when the Nominatim reverse geocode returns neither `city` nor `country`. The label `"${FACILITY_TYPE_LABELS[type]} near Unknown"` is then persisted to Redis + the dev file cache and flows to every client surface (map tooltip, counter dropdown, detail panel image alt). Client fallback changes alone cannot fix this — the "near Unknown" string is already baked into `facility.label` before any client code runs.

**Two-front fix:**

1. **Server filter tightening (D-03/D-05 follow-up)** — reject facilities in `normalizeWaterElement` that have neither a resolvable nearestCity within 150km nor a wikidata/wikipedia reference. These are exactly the low-signal dams the user is complaining about. Track rejections in a new `WaterFilterStats.rejections.no_city` bucket so the user can see exactly how many were removed in the DevApiStatus panel.

2. **Server label guard + client display-name helper (defense in depth)** — make `labelUnnamedFacilities` NEVER emit the string "Unknown" when Nominatim returns empty; leave the generic type label (`"Dam"`) intact. Add a client-side `getWaterFacilityDisplayName` helper as the single UI entry point for water labels, so any stale cached facilities carrying "near Unknown" labels from before the fix are also sanitized at read time.

Purpose: deliver on the phase-goal adjective "Palantir-grade" — show fewer high-quality facilities rather than a noisy list padded with placeholder labels.

Output: one new client helper file, four modified server files, four modified client files, a new `no_city` rejection bucket end-to-end, and a clean test run.
</objective>

```typescript
async function labelUnnamedFacilities(facilities: WaterFacility[]): Promise<WaterFacility[]> {
  const unnamed = facilities.filter((f) => GENERIC_LABELS.has(f.label));
  if (unnamed.length === 0) return facilities;

  const toGeocode = unnamed.slice(0, MAX_GEOCODE_COUNT);
  // ...
  for (const f of toGeocode) {
    const qLat = Math.round(f.lat * 100) / 100;
    const qLon = Math.round(f.lng * 100) / 100;
    const cacheKey = `${GEOCODE_CACHE_PREFIX}${qLat},${qLon}`;

    const cached = await cacheGetSafe<{ city?: string; country?: string; display?: string }>(
      cacheKey,
      GEOCODE_TTL_MS,
    );

    if (cached) {
      const location = cached.data;
      const place = location.city ?? location.country ?? 'Unknown'; // <-- HERE
      f.label = `${FACILITY_TYPE_LABELS[f.facilityType]} near ${place}`;
      continue;
    }

    try {
      const geo = await reverseGeocode(f.lat, f.lng);
      await cacheSetSafe(cacheKey, geo, GEOCODE_REDIS_TTL_SEC);
      const place = geo.city ?? geo.country ?? 'Unknown'; // <-- HERE
      f.label = `${FACILITY_TYPE_LABELS[f.facilityType]} near ${place}`;
    } catch (err) {
      log.warn({ err, facilityId: f.id }, 'reverse geocode failed for unnamed facility');
    }
    await new Promise((resolve) => setTimeout(resolve, 1050));
  }
  return facilities;
}
```

```typescript
import type { WaterFacility, WaterFacilityType } from '../../server/types';

/**
 * Human-readable facility type labels. Matches server's FACILITY_TYPE_LABELS
 * (server/adapters/overpass-water.ts). Phase 27.3 Plan 04 harmonizes the
 * "Desalination Plant" vs "Desalination" drift identified as IN-04 in the
 * Plan 03 deferred list.
 */
const WATER_TYPE_LABELS: Record<WaterFacilityType, string> = {
  dam: 'Dam',
  reservoir: 'Reservoir',
  desalination: 'Desalination Plant',
};

/**
 * Case-insensitive regex matching the pre-fix "near Unknown" sentinel that
 * labelUnnamedFacilities used to emit (server/routes/water.ts).  Phase 27.3
 * Plan 04 Task 1 removed the origin; this pattern exists so that any stale
 * cached facility still carrying the sentinel in its `label` field gets
 * sanitized at read time.
 *
 * Matches: "Dam near Unknown", "Reservoir near unknown", trailing whitespace.
 * Does NOT match: "Mosul Dam" (no "near Unknown"), "Dam near Kabul" (place is not "unknown").
 */
const NEAR_UNKNOWN_RE = /\bnear\s+unknown\s*$/i;

/**
 * Single source of truth for a water facility's UI display name.
 *
 * Fallback chain (first non-sentinel match wins):
 *   1. `facility.label` — unless it ends with the "near Unknown" sentinel
 *      (pre-Phase-27.3-Plan-04 cached labels)
 *   2. `"<Type> on <RiverName>"` — when `linkedRiver` enrichment resolved
 *   3. `"<Type> near <CityName>"` — when `nearestCity` enrichment resolved
 *      (mostly dead after Task 1's filter tightening, kept for stale caches)
 *   4. `"<Type> at <lat>°, <lng>°"` — last-resort coordinate-based label
 *
 * Invariant: the returned string NEVER contains the case-insensitive token
 * "unknown". This is the intended user-visible contract; grep-verifiable.
 *
 * Consumed by: WaterTooltip, useCounterData.toWaterEntity,
 * useProximityAlerts.waterToSiteLike, panelLabel.getCurrentPanelView,
 * WaterFacilityDetail (img alt).
 */
export function getWaterFacilityDisplayName(facility: WaterFacility): string {
  const typeLabel = WATER_TYPE_LABELS[facility.facilityType];

  // 1. Prefer the OSM-sourced label, but sanitize the pre-fix "near Unknown" sentinel.
  const raw = facility.label?.trim();
  if (raw && !NEAR_UNKNOWN_RE.test(raw)) {
    return raw;
  }

  // 2. River-based fallback — "Dam on Euphrates"
  if (facility.linkedRiver) {
    return `${typeLabel} on ${facility.linkedRiver.name}`;
  }

  // 3. City-based fallback — "Dam near Tehran". Note: the server-side filter
  //    (Phase 27.3 Plan 04 Task 1) already rejects facilities that lack BOTH
  //    a nearby city AND a wikidata ref, so this branch is rare but correct.
  if (facility.nearestCity) {
    return `${typeLabel} near ${facility.nearestCity.name}`;
  }

  // 4. Coordinate-based absolute last resort. Use compass-style NSEW signs.
  const latStr = `${Math.abs(facility.lat).toFixed(2)}°${facility.lat >= 0 ? 'N' : 'S'}`;
  const lngStr = `${Math.abs(facility.lng).toFixed(2)}°${facility.lng >= 0 ? 'E' : 'W'}`;
  return `${typeLabel} at ${latStr}, ${lngStr}`;
}
```
