- **Verification:** placeholder text grep = 1; null-return grep = 0; tsc clean.

#### UAT Test 7 — CLOSED (BLOCKER: "too little desal, no dams at all")

- **Previous state (UAT 2026-04-18):** Plan 04's `no_city` rule rejected ALL facility types when `!nearestCity && !hasWikiRef`, starving the dam and desalination categories that the phase goal explicitly preserves. User saw a near-empty map.
- **Plan 05 fix:** Scoped the `no_city` rule to `facilityType === 'reservoir'` only, with an additional named-priority-country exemption (`inPriority && hasName(el.tags)`). Dams and desalination facilities are never rejected by this rule. The reservoir noise reduction goal is preserved (Plan 05 still rejects unnamed non-priority reservoirs, the actual noise category).
- **Verification:** `facilityType === 'reservoir'` grep = 2; `isNamedInPriorityCountry` defined and gates the rule; 4 new tests cover dam-exempt, desal-exempt, named-priority-reservoir-exempt, and unnamed-non-priority-rejected paths; all 72 adapter tests pass.

#### UAT Test 8a — CLOSED ("reservoirs with just the generic name reservoir")

- **Previous state (UAT 2026-04-18):** When OSM elements had no `name`/`name:en`/`operator` tag, `extractLabel` returned the bare type label `"Reservoir"`/`"Dam"`/`"Desalination Plant"`. The client helper's sentinel only caught `"... near Unknown"`, so bare-type labels passed through unchanged and rendered to users.
- **Plan 05 fix:** Added `GENERIC_TYPE_RE = /^(dam|reservoir|desalination(?:\s+plant)?)$/i` (whole-string, case-insensitive) and extended the sentinel composition to skip the raw label and fall through to the river/city/coord chain when matched.
- **Verification:** `GENERIC_TYPE_RE` grep = 2; 9-test waterLabel.test.ts file passes; first test explicitly verifies `"Reservoir"` + linkedRiver=Euphrates → `"Reservoir on Euphrates"`; "Mosul Dam" preservation test passes (whole-string match correctly excludes substantive labels).

#### UAT Test 8b — CLOSED ("Hub Dam should be a dam")

- **Previous state (UAT 2026-04-18):** A facility tagged `natural=water`/`water=reservoir` named `"Hub Dam"` classified as a reservoir because the classifier was tag-only — OSM mappers commonly tag the impounded water surface as a reservoir even when the facility is a dam.
- **Plan 05 fix:** Added a one-directional post-classification override in `classifyWaterType`: if the tag-based result is `reservoir` AND the OSM `name` (or `name:en`) matches `/\bdam\b/i`, return `dam`. The override is one-directional — it never demotes dams or reclassifies desalination.
- **Verification:** `DAM_IN_NAME_RE` grep = 2 (declaration + single usage); 6 new classifier tests cover Hub Dam, Atatürk Dam, name:en preference, Damascus non-match (word-boundary correctness), dam → dam preservation, and desalination immunity; all pass.
