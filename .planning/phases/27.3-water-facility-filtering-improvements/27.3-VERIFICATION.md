### Gap Closure Confirmation (Plan 27.3-04)

#### UAT Test 3 — CLOSED ("Dam near unknown" no longer appears)

- **Previous state (VERIFICATION 2026-04-17T08:35Z → UAT retest):** User reported "Some of these dams are unnecessary… it just says 'Dam near unknown'." Root cause: `server/routes/water.ts` `labelUnnamedFacilities` wrote the literal string `'Unknown'` into `facility.label` when Nominatim returned empty, and this label got persisted to Redis + the dev cache, flowing to every client display surface.
- **Two-front fix (Plan 04):**
  1. **Server filter tightening** — `normalizeWaterElement` now rejects facilities that have neither a nearestCity within 150km nor a wikidata/wikipedia tag. Tracked in new `stats.rejections.no_city` bucket (wired end-to-end: server interface → Zod schema → test fixture → client interface → DevApiStatus UI).
  2. **Server label guard + client display-name helper** — both `labelUnnamedFacilities` branches now guard with `if (place)` before overwriting `f.label`, so the literal `'Unknown'` is gone. New `src/lib/waterLabel.ts` exports `getWaterFacilityDisplayName(facility)` which enforces a `label → linkedRiver → nearestCity → lat/lng NSEW` fallback chain and sanitizes any stale cached "near Unknown" labels via the NEAR_UNKNOWN_RE sentinel regex. All 5 client surfaces migrated: WaterTooltip, counter rows, proximity alerts, breadcrumb (panelLabel), detail-panel image alt.
