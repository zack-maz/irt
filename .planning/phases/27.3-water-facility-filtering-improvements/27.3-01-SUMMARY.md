# Plan 27.3-01 Summary

## What Was Built

Server-side foundation for water facility quality improvements with all review feedback incorporated.

## Changes

### Types & Schemas

- `WaterFacilityType` reduced from 4 members to 3 (removed `treatment_plant`)
- `WaterFacility` interface gained 4 optional enrichment fields: `capacity`, `nearestCity`, `linkedRiver`, `notabilityScore`
- `waterResponseSchema` Zod schema gained optional `filterStats` envelope field
- OpenAPI spec updated to match

### Adapter (server/adapters/overpass-water.ts)

- **D-01:** Dam query union (`waterway=dam` + `man_made=dam`)
- **D-03:** `treatment_plant` query + `classifyWaterType` case + label removed
- **D-05:** Spatial dedup (50m threshold, same facilityType)
- **REV-1:** `computeNotabilityScore` (0-100 scale) with `MIN_NOTABILITY_SCORE=25` cutoff
- **REV-2:** Reservoir wikidata fallback: `isNotable || (inPriority && hasName)`
- **REV-3:** `RIVER_BBOXES` pre-computed at module init with 0.2° margin
- **D-06:** `extractCapacityTags` with unit suffix stripping
- **D-07:** `findNearestCity` with 51-city inline array + 150km threshold
- **D-08:** `linkRiver` using bbox pre-filter, 20km threshold
- **D-04:** `WaterFilterStats` interface with rawCounts, filteredCounts, rejections breakdown (5 categories), enrichment coverage, score histogram (5 buckets)

### Cache (server/cache/devFileCache.ts)

- **REV-4:** Added `saveDevWaterCache`/`loadDevWaterCache` following LLM cache pattern (7-day TTL)

### Route (server/routes/water.ts)

- Dev file cache fallback checked before Overpass call
- `filterStats` included in non-cached response envelope
- `fetchWaterFacilities` new `{facilities, stats}` return shape destructured
- `cron-warm.ts` updated for new return shape (side-effect fix)

### Tests

- Deleted 2 failing treatment_plant tests
- Added 30+ new tests covering:
  - Dam union tag classification
  - `landuse=reservoir` deprecated tag
  - treatment_plant rejection
  - Notability score buckets
  - Capacity tag extraction with unit stripping
  - Nearest city lookup (including Tabqa REV-1 expansion)
  - River bbox validity and linkage
  - Reservoir REV-2 fallback acceptance/rejection
  - Holistic score cutoff rejection
  - Normalized facility includes notabilityScore
