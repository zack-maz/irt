## Info

### IN-01: Duplicated haversine-with-coarse-filter block across three files

**File:** `src/hooks/useWaterLayers.ts:182-199`, `src/components/detail/WaterFacilityDetail.tsx:29-49`, `src/components/counters/useCounterData.ts:291-300`
**Issue:** The "5km attacked-within-radius" loop (coarse 0.05° bounding-box prefilter → haversine) is re-implemented three times with the same constants (`COARSE_DEG = 0.05`, `ATTACK_RADIUS_KM = 5`). Now that `WATER_ATTACK_EVENT_TYPES` has been extracted as the shared source of truth for the event-type gate, the geometric predicate next to it is the obvious remaining duplication. If the radius ever changes (e.g. REV-5 follow-up to 3km or 10km), three files must be kept in sync — same class of drift that caused the original WR-01 finding.
**Fix:** Extract a `computeWaterAttackStatus(facility, events, dateEnd)` helper into `src/lib/waterAttackEvents.ts` (or a sibling `waterAttackStatus.ts`) returning `{ isAttacked, isDestroyed, attackCount, attacks }`. `WaterFacilityDetail` already has the shape — move that function verbatim and have `useWaterLayers` / `useCounterData` call it per-facility (the outer loop already lives per-file). Not required for 27.3-03 closure, but worth a future cleanup ticket.

### IN-02: Dev-only `console.debug` logs every render

**File:** `src/hooks/useWaterLayers.ts:170-179`
**Issue:** The `import.meta.env.DEV` debug log runs inside the `useMemo` callback, so it fires once per memo recompute. With events polling every 15 min and facilities/filter toggles rarely changing, volume is low — but it does allocate a new `Set` (`new Set(events.map(...))`) and an ISO string on every recompute even when developers aren't looking at the console. Not a bug; consider gating behind a dev flag like `localStorage.getItem('debug:water')` if the log proves noisy, or remove once REV-5 is considered validated.
**Fix:** Optional. Either remove before merging to main or gate:

```ts
if (import.meta.env.DEV && localStorage.getItem('debug:water')) {
  console.debug('[useWaterLayers] attack detection:', { ... });
}
```
