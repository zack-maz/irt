    4. Pre-compute `destroyedIds: Set<string>` inside the useMemo (add events and dateEnd to deps):
       ```typescript
       // Pre-compute destroyed set (O(facilities * destructiveEvents))
       const destructiveEvents = events.filter(
         (e) => DESTRUCTIVE_EVENT_TYPES.has(e.type) && e.timestamp <= dateEnd,
       );
       const destroyedIds = new Set<string>();
       const COARSE_DEG = 0.05;
       for (const f of facilities) {
         for (const e of destructiveEvents) {
           if (Math.abs(e.lat - f.lat) > COARSE_DEG || Math.abs(e.lng - f.lng) > COARSE_DEG) continue;
           const dLat = (e.lat - f.lat) * Math.PI / 180;
           const dLng = (e.lng - f.lng) * Math.PI / 180;
           const a = Math.sin(dLat / 2) ** 2 +
             Math.cos(f.lat * Math.PI / 180) * Math.cos(e.lat * Math.PI / 180) *
             Math.sin(dLng / 2) ** 2;
           const dist = 6371 * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
           if (dist <= 5) { destroyedIds.add(f.id); break; }
         }
       }
       ```
       NOTE: Inline haversine to avoid importing from attackStatus (which imports SiteEntity types). Keep the pattern consistent with attackStatus.ts but self-contained.

    5. Update the `getColor` accessor on the facility IconLayer:
       ```typescript
       getColor: (d: WaterFacility) => {
         if (destroyedIds.has(d.id)) return [0, 0, 0, 255] as [number, number, number, number];
         return stressToRGBA(d.stress.compositeHealth);
       },
       ```

    6. Add `events` and `dateEnd` to the useMemo dependency array alongside `isActive` and `facilities`.
