**Precision Ring (D-03):**

1. Modify `src/components/map/PrecisionRingLayer.tsx`:
   - Import `useUIStore` from `@/stores/uiStore`
   - Add `const selectedEntityId = useUIStore((s) => s.selectedEntityId);` inside the hook
   - Remove static `RING_FILL_COLOR` and `RING_LINE_COLOR` constants
   - In the ScatterplotLayer config, replace `getFillColor: RING_FILL_COLOR` with an accessor function:
     ```typescript
     getFillColor: (d: ConflictEventEntity) => {
       const isSelected = d.id === selectedEntityId;
       return [239, 68, 68, isSelected ? 102 : 13]; // 40% vs 5% opacity
     },
     ```
   - Replace `getLineColor: RING_LINE_COLOR` with:
     ```typescript
     getLineColor: (d: ConflictEventEntity) => {
       const isSelected = d.id === selectedEntityId;
       return [239, 68, 68, isSelected ? 180 : 30]; // Bold vs subtle stroke
     },
     ```
   - Add `updateTriggers` entries for the new accessors:
     ```typescript
     updateTriggers: {
       getRadius: [ringEvents],
       getFillColor: [selectedEntityId],
       getLineColor: [selectedEntityId],
     },
     ```
   - The `selectedEntityId` must also be in the outer useMemo dependency array. Add it to the dependency array of the useMemo that creates the ScatterplotLayer.

**Zoom Icon Fix (D-04):**

2. Diagnose the zoom icon disappearance in `src/hooks/useEntityLayers.ts`:
   - The most likely cause is `billboard: false` combined with terrain perspective at high zoom. When icons are flat on the terrain surface and you zoom in close, they can become edge-on or clip through terrain.
   - Check the event IconLayer configurations. Look for `billboard: false` on event layers.
   - Fix: Set `billboard: true` for ALL event type IconLayers (airstrike, on_ground, explosion, targeted, other). Event icons do not need heading rotation (unlike flights), so billboard mode is correct for them. Flights/ships should KEEP `billboard: false` since they use heading rotation.
   - Also increase `sizeMaxPixels` for event types to prevent capping at high zoom. In `src/components/map/layers/constants.ts`, increase event maxPixels values:
     - airstrike: 160 -> 200
     - on_ground: 120 -> 160
     - explosion: 160 -> 200
     - targeted: 140 -> 180
     - other: 120 -> 160
   - If the event layers are built in useEntityLayers.ts directly (not via constants), make the billboard change there. Search for where `billboard: false` is set on event icon layers and change to `billboard: true`.

3. If icons are still disappearing after the billboard fix (verify by testing at max zoom in the browser), the secondary fix is to increase the icon atlas canvas resolution. The current atlas may be 128x128 or 256x256. If icons look blurry at high maxPixels, increase atlas to 512x512. Check `src/components/map/layers/icons.ts` for the atlas generation.
