# Phase 27.2: Event Quality and Water Data Improvements

## Scope

Improve event data quality through tiered news source filtering and richer LLM enrichment. Fix precision ring UX, zoom icon disappearance, and date slider styling. Expand water facility coverage via Overpass query tuning, add full filter parity, and increase icon sizing.

## Decisions

### 1. News Source Tiering System

**Decision:** Three-tier system with filtering + severity weighting.

- **Tier 1 (gold):** Wire services + OSINT — Reuters, AP, AFP, Bellingcat, Liveuamap. Always shown, highest severity weight.
- **Tier 2 (silver):** Major outlets — BBC, Al Jazeera, CNN, Times of Israel, Middle East Eye, Guardian, NYT, Washington Post. Shown by default, normal weight.
- **Tier 3 (bronze):** Regional/niche — Tehran Times, IRNA, SANA, Press TV. Shown but flagged as state media/partisan. Lower weight.
- **Everything else:** Filtered out entirely from news feed. Still counts toward event numSources for corroboration.
- **Minimum sources threshold:** Events with only 1 Tier 3 source get demoted. Events with 2+ Tier 1/2 sources get boosted.

### 2. LLM Enrichment — Richer Input + Smaller Batches

**Decision:** Better input quality AND smaller batches for deeper analysis.

- Feed richer source text (article titles, snippets) from high-tier sources into LLM prompts
- Drop batch size from 8 groups/call to 4 groups/call for more attention per group
- Keep existing extraction schema (no new output fields)
- With fewer junk events post-filtering, total API calls may decrease despite smaller batches

### 3. Precision Ring UX

**Decision:** Always visible at 5% opacity, bold on click to 40%.

- Rings render at 5% opacity ambient (subtle spatial uncertainty overview)
- Clicking/selecting an event makes its ring 40% opacity
- Non-selected rings stay at 5%
- Provides "uncertainty heatmap" at a glance without map clutter

### 4. Zoom Icon Disappearance Fix

**Decision:** Diagnose and fix — event entity icons should NOT disappear when zooming in.

- Likely a `sizeMaxPixels` cap issue or a deck.gl visibility threshold
- Must work at all zoom levels

### 5. Date Range Slider Styling

**Decision:** Match width and style of other double-ended sliders (altitude, speed, mentions).

- Same component pattern as RangeSlider
- Consistent visual appearance in FilterPanelSlot

### 6. Water Overpass Query — Remove Name Filter + Reverse Geocode

**Decision:** Remove `["name"]` filter to get full facility coverage, reverse geocode unnamed facilities.

- Current query requires facilities to have OSM name tag → misses most unnamed dams/treatment plants
- Remove name filter to get all tagged facilities
- For unnamed facilities, use Nominatim reverse geocode to generate labels (e.g. "Dam near Tabriz")
- Nominatim geocoder already wired up and cached from Phase 27

### 7. Water Filter Parity

**Decision:** Full filter section matching other entities.

- **Facility type toggles** — already done in quick task (dam, reservoir, desalination, treatment_plant)
- **Name search** — text filter on facility name
- **Stress level toggles** — High/Medium/Low composite health (from WRI Aqueduct)
- **Attacked/Healthy toggles** — same pattern as Sites section (SliderToggle matching Sites)

### 8. Water Facility Icon Sizing

**Decision:** Make water facility icons bigger.

- Current sizing likely too small relative to other entities
- Increase to match or approach site icon sizing

### 9. Ships Button Label

**Decision:** Rename "Ships" FilterButton to "All Ships" matching "All Flights" and "All Events" master button style.

## Deferred Ideas

- Source tier UI indicator in news feed cards (showing gold/silver/bronze badge) — could add later
- Per-source tier filtering toggle in UI (let user hide Tier 3) — future enhancement
- Water facility capacity data from OSM — most ME facilities don't have this tag
