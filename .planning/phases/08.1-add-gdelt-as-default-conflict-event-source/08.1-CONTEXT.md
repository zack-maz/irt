# Phase 08.1: Add GDELT as Default Conflict Event Source - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning
**Source:** Conversation context (GDELT API research + user requirements)

<domain>
## Phase Boundary

Replace ACLED as the default conflict event data source with GDELT v2 events export. GDELT is free, requires no auth, and updates every 15 minutes. Keep ACLED adapter code intact for future use but do not expose a UI toggle to switch between them — the app just uses GDELT.

The existing eventStore, useEventPolling, event routes, and entity layers remain. Only the server-side adapter and route logic change.

</domain>

<decisions>
## Implementation Decisions

### Data Source
- GDELT v2 events export is the default (and only active) conflict event source
- ACLED adapter code stays in the codebase but is not called by the events route
- No UI toggle for switching event sources — display only shows "N events" in StatusPanel
- GDELT requires no API key or authentication

### GDELT API Pattern
- Endpoint: `http://data.gdeltproject.org/gdeltv2/lastupdate.txt` returns URLs for latest 15-min CSV exports
- Download the `.export.CSV.zip` file, unzip in memory, parse tab-delimited
- Filter by CAMEO EventCode (18x=Assault, 19x=Fight, 20x=Unconventional mass violence) AND ActionGeo_CountryCode (Middle East FIPS codes)
- Key columns (1-indexed, tab-delimited): 1=GlobalEventID, 2=SQLDATE, 27=EventCode, 28=EventBaseCode, 31=GoldsteinScale, 32=NumMentions, 53=ActionGeo_FullName, 54=ActionGeo_CountryCode, 57=ActionGeo_Lat, 58=ActionGeo_Long, 61=SOURCEURL

### Middle East FIPS Codes
- IR=Iran, IZ=Iraq, SY=Syria, TU=Turkey, SA=Saudi Arabia, YM=Yemen, MU=Oman, AE=UAE, QA=Qatar, BA=Bahrain, KU=Kuwait, JO=Jordan, IS=Israel, LE=Lebanon, AF=Afghanistan, PK=Pakistan

### Polling & Caching
- Poll every 15 minutes (matches GDELT update cycle)
- Update useEventPolling interval from 300s (ACLED) to 900s (15 min)
- Server-side cache with 15-minute TTL
- No stale clearing for events (same as ACLED — historical data)

### Event Classification
- Map CAMEO codes to existing entity types: drone strikes → 'drone', artillery/missile → 'missile'
- EventCode 190-195 = conventional military force → classify by sub-code or default to 'missile'
- EventCode 18x = assault → classify as 'drone' (closest match)
- EventCode 20x = unconventional mass violence → classify as 'missile'

### Claude's Discretion
- How to handle CSV parsing (streaming vs full load — files are ~86KB, full load is fine)
- Deduplication strategy if needed (GDELT can have duplicate events from different news sources)
- Whether to use a ZIP library or Node's built-in zlib

</decisions>

<specifics>
## Specific Ideas

- GDELT exports are small (~86KB zipped) — download and parse entirely in memory
- Use Node's built-in zlib for decompression, no external dependency needed
- The GoldsteinScale column (col 31) indicates conflict intensity (-10 to +10, negative = conflict) — could be used for severity
- NumMentions (col 32) indicates how widely reported an event is — could help filter noise
- SOURCEURL (col 61) could be stored for future "click to see source" feature

</specifics>

<deferred>
## Deferred Ideas

- UI toggle to switch between GDELT and ACLED (wait until ACLED access is approved)
- Event severity/intensity visualization using GoldsteinScale
- Deduplication by location proximity + time window
- Click-to-source linking from map entity to news article

</deferred>
