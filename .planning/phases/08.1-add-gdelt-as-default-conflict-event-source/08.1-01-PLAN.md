Install adm-zip for in-memory ZIP decompression and its TypeScript types:
```bash
npm install adm-zip && npm install -D @types/adm-zip
```

Do NOT use Node.js built-in zlib -- it handles gzip/deflate compression, NOT ZIP archives. ZIP is a container format with its own header structure. adm-zip provides a simple synchronous buffer-in/buffer-out API suitable for the ~86KB GDELT export files.

**GREEN phase:** Create `server/adapters/gdelt.ts` implementing:

1. Constants:
   - `GDELT_LASTUPDATE_URL = 'http://data.gdeltproject.org/gdeltv2/lastupdate.txt'` (HTTP, NOT HTTPS -- GDELT has TLS cert issues)
   - `MIDDLE_EAST_FIPS` Set with 16 FIPS 10-4 codes: IR, IZ, SY, TU, SA, YM, MU, AE, QA, BA, KU, JO, IS, LE, AF, PK
   - `CONFLICT_ROOT_CODES` Set with: '18', '19', '20'
   - `COL` object with 0-indexed column positions per RESEARCH.md Pattern 3

2. `getExportUrl()`: Fetch lastupdate.txt, find line containing '.export.CSV.zip', return the URL (third space-delimited field)

3. `downloadAndUnzip(url)`: Fetch ZIP, convert to Buffer, use AdmZip to decompress, return CSV text from first entry

4. `classifyByCAMEO(eventRootCode, eventCode)`: 18->'drone', 19/20->'missile' per CONTEXT.md

5. `parseSqlDate(sqlDate)`: Parse YYYYMMDD string to Unix ms timestamp using `new Date(year, month-1, day).getTime()`

6. `describeEvent(eventRootCode)`: Return human-readable label: '18'->'Assault', '19'->'Armed conflict', '20'->'Mass violence'

7. `normalizeGdeltEvent(cols, lat, lng)`: Build ConflictEventEntity with:
   - id: `gdelt-${cols[COL.GLOBALEVENTID]}`
   - type: from classifyByCAMEO
   - label: `${ActionGeo_FullName}: ${describeEvent(rootCode)}`
   - data.fatalities: 0 (GDELT does not track fatalities)
   - data.source: cols[COL.SOURCEURL] or empty string
   - data.actor1, actor2, notes: empty strings

8. `parseAndFilter(csv)`: Split by newline, split each by tab, filter by column count >= 61, filter by EventRootCode in CONFLICT_ROOT_CODES, filter by CountryCode in MIDDLE_EAST_FIPS, filter by valid lat/lng (not NaN), normalize survivors

9. `fetchEvents()`: Orchestrate getExportUrl -> downloadAndUnzip -> parseAndFilter. Log count and timing like ACLED adapter does.

CRITICAL anti-patterns to avoid:
- Do NOT use HTTPS for GDELT URLs (TLS cert issues)
- Split on '\t' not ',' (tab-delimited despite .CSV extension)
- Use FIPS codes not ISO codes (IZ not IQ, TU not TR, IS not IL)
- Use EventRootCode (col 28) not EventCode for filtering
- Guard against NaN lat/lng

Run tests -- they MUST pass.
