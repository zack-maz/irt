## Knip Whitelisted Entries (with rationale)

- **`tailwindcss`** — used via `@import 'tailwindcss'` in `src/styles/app.css`. Knip's CSS parser does not pick this up.
- **`pino-pretty`** — used at runtime by `pino` in dev mode via `target: 'pino-pretty'` string literal in `server/lib/logger.ts`. Knip cannot follow string-literal references.
- **`@types/pino-http`** — type-only package consumed transitively by `pino-http`'s augmentation of Express's Request type. Knip flags type-only deps when no `import type` exists.

## Decisions Made

See `key-decisions` in frontmatter. The two most consequential ones:

1. **Treat pre-existing lint errors as in-scope.** When the cleanup task started, `npm run lint` was already failing with 81 errors (pre-existing tech debt from earlier phases). The task's verify block requires lint to exit 0, so I fixed all 81 even though they were not directly introduced by this plan. Without this, downstream plans (CI, pre-commit hooks) would inherit a broken baseline.
2. **Drift check uses dynamic import + NODE_ENV=test.** `server/config.ts` calls `parseEnv()` at module load and crashes if Redis credentials are missing. The drift checker forces `NODE_ENV=test` before the dynamic import so `parseEnv()` returns safe test defaults. This lets contributors run `npm run check:env` locally without setting up real Redis.

## Issues Encountered

- **eslint `prefer-const` flagged at the assignment site, not the declaration.** In `server/adapters/aisstream.ts`, `let timer` is declared at line 30 but assigned at line 92 (inside a callback closure). ESLint reports the error at line 92, not 30, which makes the disable comment placement counter-intuitive. Resolved with an inline `eslint-disable-next-line prefer-const` on the assignment line plus a multi-line comment at the declaration explaining why `let` is required (the error handler closes over `timer` before the setTimeout runs).
- **maplibre 5 `color-relief` layer type missing from `@vis.gl/react-maplibre` 8.x.** Casting `type` and `paint` together with `as 'raster'` produced cascading errors because `'raster'` doesn't accept `color-relief-*` paint properties. Resolved by isolating the cast in a `ColorReliefLayer` wrapper component using a single `as any` with a documented eslint-disable comment.
