# Quick Task 260411-m00: Fix threat density cluster sizes

## What changed

Threat density clusters now scale visually with event count instead of rendering at a fixed 60px size for all clusters.

### Before

- `radiusUnits: 'meters'` with `radiusMinPixels: 60, radiusMaxPixels: 60` — all clusters identical size
- `getRadius: 100_000` — static value, no event count dependency

### After

- `radiusUnits: 'pixels'` — zoom-independent (pixel sizes don't change with zoom level)
- `getRadius: 25 + sqrt(eventCount) * 20` — scales with event count, diminishing returns via sqrt
- `radiusMinPixels: 25, radiusMaxPixels: 200` — reasonable bounds

### Color (unchanged)

Color is determined by threat weight normalized against P90:

- `computeThreatWeight`: typeWeight x log2(mentions) x log2(sources) x fatalityFactor x goldsteinHostility
- 4-stop thermal palette: deep purple (low) -> magenta -> orange -> bright red (extreme)
- P90 normalization ensures top 10% of clusters always render at maximum color intensity
