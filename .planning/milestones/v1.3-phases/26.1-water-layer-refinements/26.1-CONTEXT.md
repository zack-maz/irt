<decisions>
## Implementation Decisions

### Data Source & Caching

- **Pure Overpass** — no static seed file, Overpass is the sole source
- Server fetches on startup, caches to Redis 24h logical TTL (7-day hard TTL for stale survival)
- **Vercel cron** hits `/api/water?refresh=true` once daily to refresh cache
- If refresh fails, serve stale cached data, retry next day
- Remove the current `Promise.race` route timeout — let the cron handle refresh timing

### Facility Types & Tiered Coverage (~1000 target)

- **Priority countries** (IL, JO, LB, SY, IQ, IR, AF): query ALL named dams, reservoirs, treatment plants, desalination
- **Other countries** (Gulf states, SE Turkey, Egypt, Pakistan, Turkmenistan, etc.): query only Wikipedia/Wikidata-tagged dams, reservoirs, desalination — NO treatment plants
- Geographic exclusion still applies: western Turkey (>600km from Diyarbakir), Uzbekistan, Tajikistan, Kyrgyzstan, Kazakhstan
- `WaterFacilityType` adds `treatment_plant` back (was removed — now returns for priority countries only)
- Desalination exempt from any notability filter — keep all

### Stress Scoring

- **Keep current formula**: sqrt curve (`Math.sqrt(1 - bwsScore/5)`) + ±0.15 precipitation modifier from Open-Meteo
- **Scale 0-10** (expanded from 1-10):
  - 0: Destroyed (blacked out) — triggered by destructive GDELT events within 5km
  - 1-2: Extreme Stress
  - 3-4: High Stress
  - 5-6: Moderate
  - 7-8: Good
  - 9-10: Healthy
- **Score 0 criteria**: only destructive event types (airstrikes, bombings, shelling) within 5km — Claude picks qualifying CAMEO codes. Not protests, abductions, or general conflict.
- **Precipitation verification**: add tests confirming precipitation modifier actually flows through and changes composite scores

### Desalination Audit

- Claude researches major Gulf desalination plants (Saudi, UAE, Kuwait, Qatar, Bahrain, Oman, Israel) and cross-references against Overpass query results
- Flag any known plants missing from Overpass — report only, no manual addition
- No static fallback — pure Overpass for desalination like everything else

### Facility Markers (new 32x32 canvas icons)

- **Dam**: trapezoid wall shape (classic dam cross-section, wide base narrow top)
- **Reservoir**: water pool shape (Claude designs)
- **Treatment plant**: Claude designs (distinct from others)
- **Desalination**: factory + droplet shape (Claude designs)
- All drawn white on canvas atlas (mask mode, tinted via `getColor`)
- **Uniform size** for all facilities (no size-by-importance)
- **Score 0 (Destroyed)**: solid black fill — no tinting, visually distinct from lowest blue scores
- Add new icons to the existing icon atlas canvas in `icons.ts`

### Claude's Discretion

- Specific CAMEO codes qualifying as "destructive" for score 0
- Reservoir, treatment plant, and desalination icon designs at 32x32
- Overpass query splitting strategy to avoid rate limits (per-type bbox queries worked well in Phase 26)
- Whether to query priority vs non-priority countries in separate batches or single query with post-filter
- Treatment plant icon shape

</decisions>
