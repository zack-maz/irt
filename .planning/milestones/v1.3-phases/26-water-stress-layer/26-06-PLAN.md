  <name>Task 1: Fix API reliability and color visibility for water facilities</name>
  <files>server/routes/water.ts, server/adapters/overpass-water.ts, src/lib/waterStress.ts</files>
  <action>
  Three fixes targeting the "no facility markers visible" UAT gap (test 5):

**A) Route-level timeout in server/routes/water.ts:**
The GET `/api/water` handler currently calls `fetchWaterFacilities()` with no route-level timeout. When Overpass hangs (23-country query frequently does), the request never responds.

Wrap the `fetchWaterFacilities()` call in a Promise.race with a 30-second timeout. If the timeout fires:

- If stale cache exists, return it (already handled in catch block)
- If no cache, return `{ data: [], stale: true, lastFresh: 0 }` (empty array, not an error)
- Log a warning: `[water] Overpass timed out at route level, returning empty`

This ensures the client always gets a response. The empty array means no facility markers initially, but the next request (with cache populated by a background process or retry) will succeed.

Implementation pattern:

```typescript
const ROUTE_TIMEOUT_MS = 30_000;

// In the try block, replace the bare fetchWaterFacilities() call:
const facilities = await Promise.race([
  fetchWaterFacilities(),
  new Promise<never>((_, reject) =>
    setTimeout(() => reject(new Error('Route timeout')), ROUTE_TIMEOUT_MS),
  ),
]);
```

**B) Reduce Overpass query scope in server/adapters/overpass-water.ts:**
The current query searches 23 countries at once, which frequently times out. Split into two sequential batches of countries to reduce per-request load:

- Core batch (12 countries): IR, IQ, SY, LB, IL, PS, JO, SA, AE, KW, BH, QA
- Extended batch (11 countries): YE, TR, EG, AF, PK, AM, AZ, GE, TM, OM, DJ

Modify `fetchWaterFacilities()` to run the core batch first, then the extended batch. Each batch gets its own Overpass query. If the core batch succeeds but the extended batch fails, return the core results (partial data is better than none). Each individual batch should use the existing primary/fallback URL pattern.

Also reduce the Overpass `[timeout:]` from 120 to 60 for each smaller batch (since the scope is halved).

**C) Raise color floor in src/lib/waterStress.ts:**
The current color ramp starts at pure black `[0,0,0]` for health=0. Since 63% of ME basins have extreme stress (bws_score >= 4), most facility icons render near-black and are invisible on the dark terrain map.

Change `STRESS_COLORS[0]` from `[0, 0, 0]` (black) to `[40, 20, 60]` (dark purple). This provides enough contrast to be visible on dark terrain while still clearly reading as "stressed." Update the corresponding legend stop from `#000000` to `#28143c`.

Updated color stops:

```typescript
export const STRESS_COLORS: [number, number, number][] = [
  [40, 20, 60], // health=0: dark purple (extreme stress) -- visible on dark terrain
  [30, 58, 138], // health=0.33: dark blue
  [59, 130, 200], // health=0.66: medium blue
  [125, 211, 252], // health=1.0: light blue (healthy)
];
```

Also update `WATER_STRESS_LEGEND_STOPS[0].color` from `'#000000'` to `'#28143c'`.
