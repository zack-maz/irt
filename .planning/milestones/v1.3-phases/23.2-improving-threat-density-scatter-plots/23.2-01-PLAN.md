    **1. Create `RadialGradientExtension.ts`** (new file):
    - Import `LayerExtension` from `@deck.gl/core`
    - Create class extending LayerExtension with `static extensionName = 'RadialGradientExtension'`
    - Implement `getShaders()` returning `{ inject: { 'fs:DECKGL_FILTER_COLOR': ... } }`
    - GLSL shader: compute `float dist = length(geometry.uv);`, apply smooth falloff `float falloff = 1.0 - smoothstep(0.0, 1.0, dist);`, multiply `color.a *= falloff;`
    - Export the class as named export

    **2. Update `ThreatHeatmapOverlay.tsx`**:

    a) **Simplified palette** — Replace 8-stop `THERMAL_COLOR_RANGE` with 4 stops:
    ```
    [80, 20, 120],     // deep purple (low threat)
    [180, 30, 100],    // magenta (moderate)
    [230, 120, 30],    // orange (high)
    [255, 40, 30],     // bright red (extreme)
    ```

    b) **Update legend registration** — Change first colorStop hex to match new palette's first stop (`#501478` for deep purple) and last stop (`#ff281e` for bright red).

    c) **Pixel-based radius from cell count** — Replace meter-based `getRadius` with pixel-based sizing:
    - Change `radiusUnits` from `'meters'` to `'pixels'`
    - Remove `radiusMinPixels` and `radiusMaxPixels` (not needed with pixel units)
    - `getRadius: (d: ThreatCluster) => { const cellCount = d.cells.length; const MIN_PX = 12; const MAX_PX = 100; const t = Math.min(1, (cellCount - 1) / 19); return MIN_PX + t * (MAX_PX - MIN_PX); }`
    - This is LINEAR interpolation per user decision. Single-cell = 12px, 20+ cells = 100px.

    d) **Color encodes weight (decoupled from radius)** — Update `getFillColor`:
    - Compute `t = Math.min(1, d.totalWeight / p90Weight)` where `p90Weight = computeP90(clusters.map(c => c.totalWeight))`
    - Use P90 normalization (retained from Phase 23)
    - Interpolate across 4-stop palette: `idx = Math.min(3, Math.floor(t * 4))`
    - Base alpha = 180 (will be modulated by shader gradient)

    e) **Hover dimming** — Accept `hoveredClusterId` from the hook's parameters:
    - Change `useThreatHeatmapLayers` signature to accept an optional `hoveredClusterId: string | null` parameter
    - In `getFillColor`, if `hoveredClusterId != null`:
      - Hovered cluster: alpha = 255 (brighten)
      - Non-hovered clusters: alpha = 102 (40%)
    - If `hoveredClusterId == null`: alpha = 180 (default)
    - Add `hoveredClusterId` to the `useMemo` dependency array
    - Add `updateTriggers: { getFillColor: hoveredClusterId }` to force re-evaluation when hover changes

    f) **Additive blending** — Add `parameters` prop to ScatterplotLayer:
    ```typescript
    parameters: {
      depthWriteEnabled: false,
      blendColorSrcFactor: 'src-alpha',
      blendAlphaSrcFactor: 'src-alpha',
      blendColorDstFactor: 'one',
      blendAlphaDstFactor: 'one-minus-src-alpha',
      blendColorOperation: 'add',
      blendAlphaOperation: 'add',
    },
    ```

    g) **Attach RadialGradientExtension** — Add `extensions: [new RadialGradientExtension()]` to the ScatterplotLayer props.

    **3. Update tests in `ThreatHeatmapOverlay.test.tsx`**:
