    **1. Create `server/adapters/nominatim.ts`** (new file):
    - Export `interface GeocodedLocation { city?: string; country?: string; display?: string }`
    - Export `async function reverseGeocode(lat: number, lon: number): Promise<GeocodedLocation>`
    - Quantize coordinates to 2 decimal places: `Math.round(lat * 100) / 100` (reduces unique keys, ~1km precision)
    - Fetch from `https://nominatim.openstreetmap.org/reverse?format=jsonv2&lat=${qLat}&lon=${qLon}&zoom=10&accept-language=en`
    - Set `User-Agent: 'IranConflictMonitor/1.0 (personal project)'` header
    - Parse response: `city = data.address?.city ?? data.address?.town ?? data.address?.village`, `country = data.address?.country`, `display = data.display_name`
    - Handle HTTP errors gracefully (return `{ display: 'Unknown location' }` on failure)
    - Do NOT handle caching here -- that's the route's job

    **2. Create `server/routes/geocode.ts`** (new file):
    - Import `Router` from express, `cacheGetSafe`/`cacheSetSafe` from `../cache/redis.js`, `reverseGeocode` from `../adapters/nominatim.js`
    - Constants: `GEOCODE_CACHE_PREFIX = 'geocode:'`, `GEOCODE_TTL_MS = 30 * 24 * 60 * 60 * 1000` (30 days logical), `GEOCODE_REDIS_TTL_SEC = 90 * 24 * 60 * 60` (90 days hard)
    - Single GET route `/`:
      - Parse `lat` and `lon` from query params, validate as numbers, return 400 if invalid
      - Quantize to 2 decimal places for cache key: `geocode:${qLat},${qLon}`
      - Check cache first via `cacheGetSafe`
      - On cache hit: return `res.json(cached)` (CacheResponse format: `{ data, stale, lastFresh }`)
      - On cache miss: call `reverseGeocode(qLat, qLon)`, cache result, return `{ data: result, stale: false, lastFresh: Date.now() }`
    - Export `geocodeRouter`

    **3. Register route in `server/index.ts`**:
    - Add import: `import { geocodeRouter } from './routes/geocode.js';`
    - Add route registration after weather route: `app.use('/api/geocode', rateLimiters.geocode, cacheControl(86400, 86400), geocodeRouter);`
    - The long cache-control (1 day) is appropriate since geographic names don't change

    **4. Add rate limiter in `server/middleware/rateLimit.ts`**:
    - Add `geocode: createRateLimiter(10, 60)` to `rateLimiters` object (10 requests/minute -- low volume, mostly cache hits)

    **5. Create `server/__tests__/routes/geocode.test.ts`** (new file, Wave 0 gap):

    - Test: "returns cached result on cache hit" -- mock cacheGetSafe to return data, verify Nominatim NOT called

    - Test: "quantizes coordinates to 2 decimal places in cache key" -- verify cache key uses rounded values
    - Follow existing test patterns from `server/__tests__/routes/` directory
