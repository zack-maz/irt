2. Unzips using adm-zip (already installed)
3. Parses the TSV file (GeoNames columns: 0=geonameid, 1=name, 2=asciiname, 4=latitude, 5=longitude, 8=country_code, 14=population)
4. Filters to ME bbox (lat 0-50, lng 20-80) matching the extended political overlay bbox
5. Filters to population >= 50,000 (per user decision) to avoid ambiguous small-town names
6. Excludes city names that are common English words or < 4 characters to prevent NLP false matches. Maintain a small EXCLUDE_NAMES set for known problematic names (e.g., 'Sur', 'Salt', 'Hit', 'Hail', 'As', 'Ad', 'Al')
7. Outputs a typed JSON array to `src/data/me-cities.json`

Output shape per entry:

```typescript
interface MeCity {
  name: string; // GeoNames name field
  asciiName: string; // GeoNames asciiname field
  lat: number;
  lng: number;
  countryCode: string; // ISO 2-letter
  population: number;
}
```

Sort by population descending. Target 100-250 cities. Log summary: total downloaded, filtered count, countries represented.

Verify the output file exists and has the expected shape.

Extend `server/lib/nlpExtractor.ts`:

1. Import me-cities.json at the top: `import ME_CITIES from '../../src/data/me-cities.json';` (TypeScript resolveJsonModule is already enabled)

2. Build static lexicon at module load (executes once per cold start):

```typescript
const ME_LEXICON: Record<string, string> = {};
for (const city of ME_CITIES) {
  ME_LEXICON[city.name.toLowerCase()] = 'Place';
  if (city.asciiName && city.asciiName.toLowerCase() !== city.name.toLowerCase()) {
    ME_LEXICON[city.asciiName.toLowerCase()] = 'Place';
  }
}
```

3. Build city coordinate lookup Map (name lowercase -> { lat, lng, countryCode }). For ambiguous names (same name in multiple entries), store all variants and provide a resolver function.

4. Export `NlpExtraction` interface:

```typescript
export interface NlpExtraction {
  actors: string[];
  places: string[];
  triple: ArticleTriple;
}
```

5. Export `extractActorsAndPlaces(title: string): NlpExtraction` that:
   - Creates `nlp(title, ME_LEXICON)` (single parse pass per user decision)
   - Extracts `.places().out('array')` for place names
   - Extracts `.people().out('array')` for actor names
   - Also extracts triple via existing `extractTriple()` for backward compat
   - Returns all three in one object

6. Export `lookupCityCoords(placeName: string, originalLat?: number, originalLng?: number): { lat: number; lng: number; countryCode: string } | null` that:
   - Looks up by lowercase name in the Map
   - If multiple cities share the same name and originalLat/originalLng provided, returns the nearest one (haversine distance)
   - If no originalLat/originalLng, returns the most populous match
   - Returns null if not found

All existing extractTriple functionality MUST remain unchanged -- this is purely additive.
